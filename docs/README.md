# Documentation

This directory contains the technical documentation for the current version of the ADHD-200 dynamic functional connectivity pipeline.

The documentation focuses on the implemented methodology, software architecture, validation process, computational optimizations, and known limitations of the project. Its purpose is to facilitate maintenance, reproducibility, and future extensions of the pipeline.

The documents describe the current implementation only. Historical development decisions and previous repository versions are intentionally omitted.

## Documentation Structure

| Document | Description |
|----------|-------------|
| `architecture.md` | High-level description of the software architecture and data flow. |
| `methodology.md` | Summary of the methodological decisions implemented in the pipeline. |
| `validation.md` | Record of the validations and verification procedures performed on the current implementation. |
| `performance.md` | Computational optimizations implemented to improve execution efficiency. |
| `limitations.md` | Current methodological and technical limitations of the project. |
| `auditoria-metricas.md` | Audit of the manuscript figures against the versioned results. |
| `modificaciones_eventanado.md` | Integration notes for the windowing changes. |

## Intended Audience

This documentation is intended for:

- Researchers interested in reproducing the experiments.
- Developers extending the pipeline.
- Reviewers evaluating the implementation.
- Contributors maintaining the repository.

## Scope

The documentation complements the source code and the accompanying manuscript. It summarizes the most relevant implementation decisions without describing the internal details of each function or class.
